# 하루 할 일 — 설치형 웹앱(PWA)

프레임워크·빌드 도구 없이 파일 몇 개로 동작하는 개인용 할 일 앱입니다.
휴대폰 홈 화면에 설치할 수 있고, 오프라인에서도 열리며, 데이터는 사용하는 기기에 저장됩니다.

## 파일 구성
- `index.html` — 앱 본체(화면·기능·저장 전부 여기 들어있음). **수정은 보통 이 파일만 바꾸면 됩니다.**
- `manifest.webmanifest` — 앱 이름·아이콘·색상 정보(거의 바꿀 일 없음)
- `sw.js` — 오프라인 캐시용 서비스워커(거의 바꿀 일 없음)
- `icon-192.png`, `icon-512.png`, `icon-maskable-512.png` — 홈 화면 아이콘

## GitHub Pages로 올리기(무료)
1. GitHub에서 새 저장소(repository)를 만듭니다. 예: `daily-tasks`
2. 이 폴더의 파일들을 저장소 최상단에 올립니다(드래그 앤 드롭 업로드 가능).
3. 저장소 **Settings → Pages** 로 갑니다.
4. **Source** 를 `Deploy from a branch`, 브랜치를 `main`, 폴더를 `/ (root)` 로 지정하고 Save.
5. 잠시 뒤 `https://<사용자이름>.github.io/daily-tasks/` 주소가 나옵니다. 이게 앱 주소예요.

> PWA는 반드시 HTTPS에서 동작합니다. GitHub Pages는 기본이 HTTPS라 그대로 됩니다.

## 휴대폰에 설치
- **아이폰(Safari)**: 주소 접속 → 공유 버튼 → "홈 화면에 추가"
- **안드로이드(Chrome)**: 주소 접속 → 메뉴(⋮) → "앱 설치" 또는 "홈 화면에 추가"

설치해두면 아이콘으로 열리고, 저장 데이터도 더 안정적으로 유지됩니다.

## 수정 반영 방법
앱을 바꾸고 싶으면 새 `index.html`로 교체(업로드)하면 됩니다.
온라인 상태에서 앱을 다시 열거나 새로고침하면 최신 내용이 반영됩니다.
드물게 예전 화면이 남으면, `sw.js`의 `const CACHE = "daily-tasks-v1";` 숫자를
`v2`, `v3` 처럼 올려서 다시 올리면 확실히 갱신됩니다.

## 저장에 대해
- 데이터는 **이 앱을 여는 그 기기·브라우저 안에만** 저장됩니다(서버 없음).
- 다른 기기나 다른 사람에게 자동으로 공유되지 않습니다. 각자 자기 기록을 씁니다.
- 브라우저의 "사이트 데이터 삭제"를 하거나 앱을 지우면 기록도 사라집니다.
- 여러 기기에서 같은 데이터를 쓰거나 친구와 공유하려면 별도의 서버/DB가 필요합니다.
